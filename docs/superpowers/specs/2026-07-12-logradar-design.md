# LogRadar — 设计规格 (Design Spec)

- **日期**: 2026-07-12
- **状态**: 设计已确认，待实现计划
- **技术基座**: Tauri + Rust 核心 + Web 前端 (React + TypeScript)
- **目标平台**: Windows / macOS 原生（Linux 产品支持，v1 CI 暂缓）

---

## 1. 概述

LogRadar 是一个**离线、高性能、跨平台**的日志查询与分析桌面工具，定位类似"Notepad++ 体验 + 日志查询"。面向后端 / SRE / 研发工程师，用于对**已下载的日志文件**做取证分析：打开海量日志、跨文件搜索、跳到解释事故的那一行。

**单一任务**: 让工程师在 GB 级日志里快速找到关键行、跨文件关联、定位根因。**不做**实时 tail / 流式接入（用户场景是下载好的日志拖进来离线分析）。

## 2. 范围 (v1)

### v1 包含
- 超大文件打开（mmap + 虚拟滚动）
- 侧栏目录树 + 顶部标签页组织多文件（树管全量、tabs 钉住常看文件快速切换）

- 多条件查询（关键词 AND/OR + 级别 + 时间 + 排除，统一为多条件）
- 跨文件聚合搜索（底部搜索面板，扁平文件行结果）
- 时间过滤（用户输入年-月-日 时:分:秒，内部归一化各时间戳格式匹配）
- 级别 / 时间戳 / JSON 语法高亮
- 书签 / 分屏对比（时间同滚，容差可自定义）
- 自动换行切换
- JSON 字段检查器（原地内联展开字段树）
- 搜索历史（底部面板内，记录完整查询，点击重跑，会话级）
- 导出过滤结果（范围 / 格式 / 列 / 目标 / 预览 / 进度）
- 工作区保存（打开的文件 + 完整查询条件）
- 暗色 / 亮色一键切换

- 自动诊断规则引擎、插件 / 脚本机制、告警 — 非场景

## 3. 功能清单（锁定）

### A. 文件与大文件处理
- **A1** 打开 GB 级超大日志，内存映射 + 虚拟滚动，不全量载入内存
- **A2** 多文件组织：侧栏目录树（按文件路径分层）+ 顶部标签页（钉住常看文件快速切换；树管全量、tabs 管活跃集）
- **A3** ~~文件跟随 tail -f~~ — **移除**
- **A4** 拖拽打开 / 最近文件列表；支持压缩包 `.gz` `.zip` 拖拽，自动解压或基于压缩文件直接搜索

### B. 查询与过滤
- **B1** 关键字 / 正则搜索，命中高亮
- **B2** 按时间范围过滤（输入 `年-月-日 时:分:秒`，引擎归一化各时间戳格式后匹配）
- **B3** 多条件组合（AND / OR）：关键词 AND/OR 显式可见；级别、排除作为多条件中的子条件，不再单列
- **B4** 跨多文件聚合查询（底部搜索面板，结果按"一个命中文件全路径一行"扁平展示）
- **校验**：搜索至少需要 1 个关键词；级别、时间可选

### C. 视图与展示
- **C1** 级别 / 时间戳 / JSON 字段语法高亮
- **C2** 书签 / 标记行
- **C3** 分屏对比（两文件并排，时间同滚，容差可自定义）
- **C4** 自动换行切换
- **C5** 结构化日志（JSON）解析，字段折叠展开（原地内联展开字段树）
- **C6** 搜索历史（底部搜索面板内）：每条记录**完整查询**（关键词表达式 + 级别 + 时间 + 排除 + AND/OR 组合），作为该条标题；点击回填全部条件并重跑；**会话级**（不跨重启；持久化由 E3 工作区承担）

### D. 导航
- **D1** 搜索命中跳转（上一处 / 下一处）
- **D2** 跳转到行号 / 跳转到时间
- **D3** 级别分布缩略图（minimap，"信号迹线"样式）

### E. 数据与导出
- **E1** 导出过滤后结果（对话框：范围 / 格式 / 包含列 / 目标 / 实时预览 / 大结果进度可取消）
- **E2** ~~统计：级别计数 / Top 模式~~ — **v1 不做（deferred）**
- **E3** 工作区保存 / 恢复（打开的文件 + 完整查询条件；可与 C6 互通：工作区保存的查询即 C6 的完整条件）

### F. 跨平台与性能
- **F1** Windows / macOS / Linux 原生运行；v1 CI 矩阵 = macOS + Windows（Linux 后加）
- **F2** 后台流式处理，搜索 / 过滤不阻塞 UI
- **F3** 多核并行加速大文件搜索（rayon）

## 4. 技术栈

**Tauri + Rust 核心 + Web 前端**（从零自研，不复用现成工具）。

选择理由：两个最强诉求是"高性能核心"与"全功能富 UI"。Rust 干性能（mmap / 流式解压 / 并行搜索 / 行索引），Web 干 UI（标签组织 / 分屏 / minimap / 语法高亮 / 虚拟滚动 / JSON 折叠）。核心与 UI 分离天然满足 F2"不阻塞"。Tauri 用系统 webview，包体小、跨平台原生。

**唯一早期验证风险**：webview 渲染超长结果列表需虚拟滚动 + Web Worker，须早期验证。

## 5. 架构

### 5.1 两层 + IPC
- **Rust 核心**（Tauri 主进程内，tokio 运行时 + rayon 线程池）：拥有所有性能敏感逻辑——文件 mmap、流式解压、行索引、多核并行搜索、压缩包处理。通过 Tauri 命令暴露给前端。
- **Web 前端**（系统 webview，React + TS）：拥有全部 UI。调用 Rust 命令，通过 Tauri 事件接收流式结果。

### 5.2 关键设计决定
1. **UI 永不全量拿文件**：前端永远不接收整个日志。打开只回元数据（编码、格式、行数估算）；虚拟滚动按需请求"第 N–M 行"小窗口，Rust 只返回那一段解码文本。GB 文件前端常驻仅"当前可见的几十行"。
2. **过滤 = 投影（result list），不是就地隐藏**：多条件过滤产生"匹配行号列表"，UI 虚拟滚动在该列表上跑（"给我结果第 K 行" → Rust 映射回原行号 → 返回文本）。原行索引不动，过滤只是其上的投影。过滤 / 搜索 / 聚合复用同一套行索引，不各自维护大列表。

### 5.3 进程 / 线程模型
- webview 渲染进程与 Rust 核心线程池分离 → UI 天然不阻塞（F2）
- 每个打开的文件 = Rust 侧一个 `FileSession`（持有 mmap / 解压器 / 行索引）
- CPU 密集并行搜索走 rayon；IO 走 tokio

## 6. 组件

### 6.1 Rust 核心
- **FileSession** — 一个打开文件的生命周期（路径 / mmap 或解压器 / 编码 / 行索引）。接口：`open`、`get_lines(range)→Vec<Line>`、`line_count`、`close`。一个标签页 / 树节点一个实例。
- **LineIndexer** — 行偏移索引，支撑随机访问。接口：`offset_of_line(n)`、`line_at(n)`。内存权衡：每行 u64 偏移 = 8B/行（1 亿行 ≈ 800MB 太大）；默认**采样索引**（每 256 行一个锚点 + 锚点内顺序扫描），~3MB/亿行，随机访问略慢可接受。按文件大小自动切换（小文件全量、大文件采样）。
- **Decompressor** (A4) — gz / zip 流式解压。两模式：①视图（建行索引供随机访问，gz 用 zran 式 压缩↔解压偏移索引）；②搜索（解压 + 查询一趟过，不建索引、不落盘）。
- **QueryEngine** — 在 FileSession 行索引（或压缩流解压流）上求值多条件查询，rayon 并行分块，产出匹配行号 result list。接口：`search(session, Query) → ResultStream`（异步、可取消）。
- **Query 类型** — 多条件查询的结构化表示（谓词：文本/正则、级别∈{}、时间∈[]、NOT；AND/OR 组合）。UI 与 Rust 的共享契约。
- **FormatDetector** — 检测行结构（时间戳格式、级别 token、是否 JSON）。**关键职责：时间戳解析归一化为 epoch**，使跨各种日志时间格式的时间过滤与分屏同滚成为可能。Rust 只检测 / 解析，不渲染。

> **TailWatcher 已删除**（tail 移除）。QueryEngine 无"新行增量求值"，仅批量求值。

### 6.2 IPC 边界
- **命令**（前端→Rust）：`open_file`、`get_lines{sessionId,range}`、`get_result_rows{sessionId,query,range}`、`search{sessionId,query}→searchId`、`cancel_search{searchId}`、`export`、`workspace_save/load/list`…
- **事件**（Rust→前端，流式）：`search_results`（分批）、`search_progress`、`indexing_progress`
- **原则**：大块结果一律分批事件流式推，绝不一次性序列化整份结果。搜索立即返 `searchId`，结果走事件流；`cancel_search` 中止；关标签页 / 切新查询自动取消在飞的搜索和索引。

### 6.3 Web 前端
- **FileTree**（侧栏目录树，A2 全量）
- **TabStrip**（顶部标签页，A2 活跃集：钉住常看文件快速切换）
- **VirtualLogView**（A1 按需拉行 + result list 投影）
- **SearchPanel**（底部，Notepad++ Find-in-Files 式：表单 + 历史 + 结果叠一起）
  - **FilterForm**（B3：关键词 AND/OR + 级别 + 时间输入）
  - **SearchHistoryNav**（C6：◀▶ + ▾ 完整查询历史，会话级）
  - **ResultList**（B4：扁平文件全路径行）
- **SyntaxHighlighter**（C1）
- **Minimap / SignalTrace**（D3）
- **SplitView**（C3：两窗格 + 时间同步槽 + 容差控制）
- **Bookmarks**（C2）、**JsonInspector**（C5 内联展开）、**AutoWrap**（C4）
- **ExportDialog**（E1）、**WorkspaceManager**（E3）
- **ThemeToggle**（暗 / 亮一键切换）

> **SearchHistory 放前端、不下沉 Rust**：会话级、纯 UI 状态（下拉 / 翻历史），YAGNI。但每条记录的是完整 Query 对象（与 Rust 共享契约），点击回填全部条件并触发 `search`。

## 7. 数据流

### 7.1 打开文件
拖拽 / 最近文件 → FileTree → `open_file(path)` → `FileSession::open`：按魔数判类型（纯文本 / gz / zip）→ 纯文本 mmap + 检测编码（UTF-8 否则 GBK 启发，照顾中文日志）→ `LineIndexer` 后台增量建索引；gz/zip 走 `Decompressor` 视图模式建行索引 + zran 偏移索引。`FormatDetector` 采样几行出高亮规格 + 时间戳格式喂前端。**打开快返回**：先回 sessionId + 编码 + 格式 + 行数估算，索引后台继续建（`indexing_progress`），随机访问渐进开放。UI 随即请求首页窗口渲染。

### 7.2 滚动渲染
滚动 → `VirtualLogView` 算可见范围（未过滤 = 原始行；已过滤 = result list 行）→ `get_lines` / `get_result_rows` → Rust 经 `LineIndexer` 映射偏移、按编码解码那一段 → 返回 `Vec<Line>`（仅此窗口）→ 渲染 + 高亮。minimap 同步视口。

### 7.3 搜索 / 过滤（核心）
`FilterPanel` 构造 `Query` → `search(sid, query)` 立即返 `searchId` → `QueryEngine` rayon 并行分块扫行索引（压缩文件扫解压流），命中行号入 result list → `search_results` 分批事件流式回 + `search_progress` → UI 累积结果、虚拟化切到 result list 坐标系。`SearchPanel` 提交同时记一笔进 C6 历史（完整 Query）。关标签页 / 切新查询 → 自动 `cancel_search`。

### 7.4 压缩搜索（A4 专项）
"搜 .gz 不解压"：`Decompressor` 流式解压 + `QueryEngine` 一趟过，只收命中行（带解压偏移）；不建全量索引、不落盘。要看某命中周边 → 该区域惰性建局部 zran 索引。**大 gz 默认"先搜后视"，小 gz 直接建索引可滚。**

### 7.5 导出（E1）
`export(sid, query?, format, columns, target)` → Rust 流式写命中行到文件（或剪贴板）、返路径。大结果走 `export_progress` + 可取消。盘满 → 中止 + 清理半成品。

### 7.6 ~~tail 跟随~~ — 已移除

## 8. 错误处理与边界

原则：**永不因坏数据崩 UI**；坏文件降级（能读多少读多少 + 标记不完整），坏查询前置校验 + 可中止。

- **文件 / 打开**：不存在 / 无权限 / 被锁 → 命令返明确错误，不建标签；打开中途文件消失 → 降级只读已读部分 + 标记；网络盘（SMB/NFS）mmap 慢 / 失败 → 回退普通 read + 预读，提示"性能受限"；二进制当日志 → 乱码不崩，提示"疑似非文本"。
- **编码**：非 UTF-8 启发判 GBK；混合 / 非法字节 → lossy 解码（替换符），绝不崩；BOM 正确处理。
- **压缩（A4）**：截断 gz → 解到能解的，标记不完整；gz 中段损坏 → 报错点、重同步到下一 member 边界继续；加密 / 带密码 zip → 明确"无法解压"；多 entry zip → 选 entry 或全搜。
- **查询**：非法正则 → 编译期拦、回 UI 明确、不进扫描；灾难性回溯正则 → 扫描超时 / 中止，提示"正则危险"；时间谓词但时间戳解析不了 → 跳过该谓词 + 警告；命中为空 → 空结果，非错误。
- **并发 / 资源**：搜索异步 + 可取消（`searchId` + `cancel_search`）；关标签页 / 切新查询自动取消在飞的搜索和索引 → 防僵尸工作、不阻塞命令通道。
- **结果列表内存**：极宽过滤（命中百万级）→ 默认**截断 + 警告"结果过多，请收窄过滤"**；spill-to-disk v1 不做。
- **工作区 / 导出**：工作区引用文件已不存在 → 标记缺失，不崩恢复；导出盘满 → 中止 + 清理半成品。

## 9. 测试策略

重心压在 Rust 核心（逻辑和性能都在那）。

- **Rust 单元 / 集成**（cargo test）：
  - `LineIndexer`：已知偏移合成文件、采样 vs 全量一致性、空文件 / 单行 / 无尾换行 / CRLF / 超长单行
  - `Decompressor`：gz/zip 往返、截断 gz、中段损坏重同步、多 member / 多 entry、加密 zip 报错、"搜不解压"结果 == 先解压再搜
  - `QueryEngine`：各谓词单测、AND/OR 优先级、灾难正则超时触发、空结果 / 全命中、并行分块 == 串行结果
  - `FormatDetector`：各时间戳格式（ISO8601 / syslog / epoch / `2026/07/11 …`）归一化为 epoch、级别 token、JSON 检测、多行堆栈
  - 编码：UTF-8 / GBK / 混合 / 非法字节 lossy
  - **proptest 属性测试**：`line_at(n)` 拼接 == 源文本（任意 n）；查询 AND/OR 分配律；流式解压 == 全量解压；时间归一化可逆
- **IPC 契约测试**：假前端驱动命令 / 事件面：中途取消确实停且释放、分批边界正确、indexing_progress 触发
- **前端**（Vitest + Testing Library）：`VirtualLogView` 滚动只请求可见窗口（断言命令调用）；`FilterPanel` 输出正确 Query；`SearchHistoryNav` 回填完整条件重跑；`SplitView` 同滚 + 容差逻辑；`JsonInspector` 折叠 / 展开；`ExportDialog` 预览随列变化
- **E2E**：Tauri 驱动真实 app：打开样例日志 → 搜索 → 过滤 → 导出，**macOS + Windows CI 矩阵**（Linux 后加）。GB 级留在 Rust 集成层，E2E 用小样例求快。
- **性能 / CI**：**criterion 基准**：行索引吞吐、GB 文件搜索吞吐、解压 + 搜索吞吐——防回退。**GB 级 fixture 只存生成器不存文件**；GB 性能基准 nightly / 按需跑（太慢不进每 PR）。

## 10. UI / UX 设计（premium）

### 10.1 风格方向
"精密仪器 / 信号实验室"。避开 VS Code 通用暗，也避开三大 AI 默认（暖奶油衬线 / 近黑荧光绿 / 报纸毛边）。深墨冷调底 + 单一青绿"信号"主色（雷达扫描色）+ 去饱和级别色。名字"Radar"落到具体视觉：右侧 minimap 做成"信号迹线"。

### 10.2 设计令牌
- **配色（暗）**：底 `#0B0E13` / 面 `#11151C` / 面2 `#161C25` / 边 `#232B37`；文 `#DCE3EC` / 次文 `#8A94A3` / 淡 `#565F6D`；**信号青绿 `#45C6B8`**（命中 / 活动 / 视口 / 跳转）；级别去饱和：ERR `#E8586A` / WARN `#E8A13C` / INFO `#5B9DD9` / DEBUG `#7A828D`
- **配色（亮）**：底 `#EEF0F3` / 面 `#FFFFFF` / 边 `#D9DEE5`；文 `#161B22` / 次文 `#565E6B`；**信号深青 `#1E8A7C`**；级别加深：ERR `#C93A4E` / WARN `#B5712A` / INFO `#2E6FA0` / DEBUG `#6A727C`
- **字体**：UI = **Space Grotesk**（有性格、技术感，非 Inter）；日志 / 数据 = **IBM Plex Mono**（非 JetBrains）
- **签名元素**：① 右侧 minimap = "信号迹线"（基线 + 级别尖峰 blip + 青绿扫描窗口）；② 欢迎页雷达扫描动画（暗色 = 磷光发光、亮色 = 干净线稿仪表图）；③ 工作区卡用文件路径 + 查询 chips 预览"恢复会带回什么"
- **暗 / 亮一键切换**：顶栏日 / 月按钮，CSS 变量 + `data-theme` 切换
- **克制原则**：介绍性文字克制，能用 UI 表达的就用 UI（格式徽章、分段控件、勾选、预览实样、状态色，而非句子）

## 11. 分屏同滚语义（C3 专项）

- 同滚按**归一化时间**对齐，不按行号。每个文件的时间戳经 `FormatDetector` 归一化成 epoch（与搜索时间过滤同源）。
- 锚定一侧时间 `T`，另一侧滚到时间戳最接近 `T` 的行（`min|Δ|`，默认；另可选"向前对齐"= 最大 ≤ T，滚动更稳但可能落到无关前一行）。
- 两侧时间**不一致**时，同步槽显示**两个时间 + Δ 增量**（如 `+17ms`），明示近似对齐。
- **容差用户可自定义**：默认 1s；`Δ ≤ 容差` → 青绿"对齐"；`Δ > 容差` → 琥珀"超差"（仍 snap 最近行、但标超差）；`容差 = ∞` → 永远 snap。
- 某文件无时间戳 → 该对同滚禁用、回退独立滚动并提示。
- 为减抖动：只在锚点移动超阈值时才重算对侧（滞回）。

## 12. 时间过滤语义（B2 专项）

- 用户输入 `年-月-日 时:分:秒`（可范围 `from ~ to`）。
- 引擎内部把日志里各种时间戳格式（ISO8601 / syslog / epoch 毫秒 / `2026/07/11 14:22:01` 等）经 `FormatDetector` 解析归一化为 epoch，与用户输入归一化后的 epoch 比较。
- 一份查询能跨多种日志时间格式命中；用户无需知道日志的时间格式。
- 时间戳解析不了的行 → 时间谓词跳过 + 警告（不崩）。
